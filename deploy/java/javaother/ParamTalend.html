Sheet 2: Best Practice
Model always start with Business Model any job or any projects
Folder have separate folder in business and job for use case wise or module wise
Status define the status which relevant to a project
Metadata define DB, Files and etc in metadata for a project. If those are shared across project export and reuse the same
Debug run Trace Debug rather than Java Debug. Since it need java knowledge
variable if variable used more than one COMPONENT then define context variable in that job's context view

if variable used more than one JOB then define context variable in that repositary->context

create one context group for every environment, with same set of variable name. But value for each group may be different
