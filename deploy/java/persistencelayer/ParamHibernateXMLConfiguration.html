Sheet 1: Types of config
Property file file name must be 'hibernate.properties' and place in root class path

just creation of instance to 'Configuration' is enough

Mapping file has to be add by 'addClass()' or 'addResource()' method

while use of addClass(), mapping file MUST be placed along with class location

Mapping file must be in in classpath


XML file it is best for either simple or enterprise application

file name may be in 'hibernate.cfg.xml' and place in root class path

to load 'Configuration.configure()' method has to be called

file name can be other than default, that time path of file has to be given in 'configure()' method

Mapping files can be declared in xml file itself. So no need any method call

Possible to have more than one file for different environment

mapping file can be outside of classpath


Programatic it possible, but it is not encouraged

since configuration should be externalized.

if need tight security, use this apporach to configure


JMX ( no need ) it is possilbe only in Jboss

it is easy, but not portable


Command line arg it is worst.

donâ€™t use.
