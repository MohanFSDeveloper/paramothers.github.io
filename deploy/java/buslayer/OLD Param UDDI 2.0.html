Sheet 9: Exam Key Points
tModel WS-I profile's WSDL tModel can be used for categorization only for "wsdlSpec" tModel




BindingTemplate HostingRedirector is not allowed for <bindingElement> according to WS-1 profile




Inquiry API all SOAP message are document/literal based

all are request / response message

uddi 2.0 support UTF-8, but accroding to WS-I, uddi yet to support UTF - 16 and UTF - 8

All of UDDI operation are only in <Body> of soap, and uddi is not support <Header> of SOAP




find-NamePattern nams are case-INsensitive by default

% -wild card add at the end of name by default
