7: Class & Interface
Classes

Use abstract classes, when a task has to be implement by sub class

Give proper documentation for every classes and its role

Put generic logic in super class when need more than one place

Keep your class definition highly coherent

Never prefer to write any type of inner classes except for event programming

Instantiate any object just before to use instead at the time variable declaration

Dont have more than 500 lines in a class

Never Write Constructor unless you really need , let compiler put default constructor

use this operator only inside of constructor or method only when exactly mandatory


Interface

throws clause

Use Interface to achieve loosely coupling




Main Keep your main method alone in separate class and keep its definition as minimum

11: Common Architecture
DEBUG AppDTODebuger class should have one static method for EACH type of DTO as parameter [ this looks better to code than LOG4J ]

Each Static method of AppDTODebuger should check DEBUG flag and should print the attribute of DTO

Every CATCH block should have printStrackTrace in Action and DAO classes with if check of DEBUG flag


Serialization Use Serialization if a page takes more time to display the page while doing JSP Bug Fix or JSP / Javascript testing
CODING Start the Logger while execution to know the status of execution

The methods MUST take only OBJECT-INSTANCE as parameter. Never use primitive/wrapper as parameter


Loading singleton it is wise, to load all the singleton object at the startup of application. It save time
Property file value convert to boolean get value of given key and compare with "TRUE/T/true/FALSE/F/false" and give as boolean the comparision result
exception handling instead of user getter/setter method for wrapping exception ourselves, simply rethrow when get exception at any depth layer, and use getCause method to recursively to get inner most excetion while handling excetiption




integration of any framework integration of any framework like spring, struts, ibatis and etc should be single place. Super or start class is best plac

create customized annotation wherever possible


Resuse of BusinessLogic code to know, current call of business from either webservice / JMS/ web request, use the below idea

declare one interface

declare one enum with possible value like "WEBSERVCIECALL/JMSCALL/WEBCALL"

writing one implementation class for that interface

create separate insteace variable for that same impl class with each enum constants and keep it public

when calling business add one parameter with any of one among above


Super class declare one STRING(for performance ) variable to hold class name of sub class. It will help if super class logging
