1: Variable Declaration
1 Use CamelCase naming convention for Class, Interface, Methods and Variables.
2 Proper names for class, methods and variables.
3 Prefer reference variable as Interface type in method parameter and return type.
4 Reference variable should be same as class name with first letter in lower case.
5 Use appropriate Access Specifier for variable instead of keeping them in default visibility.
6 Never do Primitive parsing and wrapping, instead use auto-boxing.
7 Use Wrapper for primitive.
8 Never declare any variable in class scope unless it is mandatory.
9 Don't declare reference or primitive variable then and there. Put all of them in beginning of method.

2: Branching
1 Give one line space after any control statement.
2 If-else block always should have { } braces even if it have a single statement.
3 Put logger to print the value before any branching.
4 Put negative validation in beginning of your method definition instead of placing it in the middle/end.
5 Use if statement in a nested manner instead of using many conditions in a single statement.
6 Switch-Case must have default block, though it is not mandatory.
7 Minimize the number of statements in case block, if more logic need then, write it in separate method and invoke that in case block.

3: Method
1 Method name should be appropriate, first letter should be lowercase.
2 Service layer method name should start like “save”, “create”, “remove”, “list”.
3 DAO layer method name should start like “delete, update, select, insert”.
4 Prefer chain method call instead of declaring many reference variable and import statements.
5 Do input parameter validation in very beginning itself and have it in a separate method.
6 Use equalIgnoreCase method to compare your command line input parameters.
7 Use appropriate Access Specifier for method instead of keeping them in default visibility.
8 Declare and use boolean variable to return appropriate flag instead of having return statement then and there.
9 In any method, there should be a single return statement at the end.
10 Entire method definition must be in a try-catch block.
11 Please don't add any business logic in catch block except handling the exception or printing stack trace.
12 Every method should have entry, exit and error logs.
13 Variable declaration must be in the beginning of methods as well as before the try statement.
14 Every public method should have proper document.
15 Keep method definition within 20 lines.

4: Loops
1 Use For-in loops for more readability.
2 Try to use For loop whenever possible instead of while and do-while.

5: Collection
1 Prefer SET to avoid duplicates.
2 Use Java Generics to avoid type casting and exceptions at run-time.
3 Don't use Vector, HashTable and Dictionary which have been depricated.
4 Use Collections API for manipulating different collection objects in JAVA.

6: Formating
1 Give proper indentation.

7: Log and Out
1 Proper usage of console “out” statement to differentiate sysout/error/debug/info/warn.
2 Give proper log message, variable name along with its value.
3 Keep print stack trace method till UAT for debugging.

8: Misc
1 Delete unused and commented source codes and files.
2 Use of System.exit() must be in one place only in your application. Should not use then and there.
3 Use finally block to close any resources like I/O, DB, Network, etc.
4 Change char count in a line to 300.
5 Format your coding, fix import in eclipse before delivery.

9: DB
1 Keep DB related information or any configuration values in Property file instead of java file itself.

10: API
1 Use Calendar class instead of creating many date object for date manipulation.
2 Use one instance of *formatter object for entire application and set its pattern before usage.
3 Usage of DateFormat, NumberFormat should be in static class.
4 Use StringBuilder instead of StringBuffer for string manipulation for single-threaded process.
5 Use String.split() method only once to tokenize a string.
