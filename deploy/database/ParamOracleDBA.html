Shared Pool


Undo segments it is used to REVERSE the effect of DML ( UPDDATE/ DELETE/ INSERT )

segment introduce at Oracle 9i

it help to achieve CONSISTENCE in ACID

it help to achieve previous image after rollback

it help to achieve ISOLATION in ACID

it used by flashback query




Shared Pool Structure

size is dynamic and change can be without restart instance, Oralce 9i onwards

dozen of different structure, few are

1. Library cache

2. Data dictionary cache

3. PL/SQL area

4. SQL Query, PL/SQL function result cache




Library cache keep recently parsed SQL code, the Prepared statement


Data Dictionary cache it keep the data dictionary row in cache for faster parsing of a query

its size managed by Oracle and most of time same size of libray cache


Pl/SQL area it is used to cache Pl/ SQL stored objects like func, proc, pack, trig, object Types

it is read from data dictionary and stored in PL/SQL cache but not in Data dictionary Cached


SQL query & PL/SQL FUNCTION Result cache

it is new in 11g, to cache query result

its size can controlled by DBA


Larger pool it is optional

its size dynamic and automatically resized

it is used by shared servers, parallel process, recovery manager




java pool it is used by java-stored procedure

its size dynamic and automatically resized




stream pool it is used by Oracle Stream

its size dynamic and automatically resized
