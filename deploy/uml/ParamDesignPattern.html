Design principles ( From Head First )


encapsulate what vaires into a separate classes ( easy to change and reuse ) startegy
Program to an interface, not implementation ( that is either abstract class / interface. Both almost same ) startegy
Favor composition over inheritance ( allow to change behavour dynamically as well as reuse)( one level subclass ok ) startegy
Strive for loosly coupled design between objects that interact each other observer
